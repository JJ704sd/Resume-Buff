# 简历帮前端 UI 优化计划

> 参考国内主流招聘网站（Boss 直聘 / 猎聘 / 智联招聘）视觉风格，对 `frontend/` 进行改版。
> 目标：更贴近「招聘 / 简历」场景的视觉语言，提升专业感与可读性，**不改动任何后端接口、不改动业务逻辑与数据流**。

## 一、设计基调（对标招聘站）

| 维度 | 现状 | 优化后（参考 Boss 直聘 / 猎聘） |
|------|------|------|
| 品牌色 | 蓝紫渐变 `#1f4e79→#2e75b6` | 招聘站主流「青绿/品牌绿」`#00bebd` 系 + 中性深灰文字；保留专业感 |
| 顶栏 | 重型渐变 + 步骤条 | 轻量顶栏：左侧 logo + 产品名，右侧极简步骤指示；白色/浅灰底 + 细描边 |
| 卡片 | 重阴影 hover | 轻阴影、统一圆角 10px、浅边框，聚焦内容 |
| 岗位选择 | 下拉 select | 「职位卡片」网格 / 标签化选择，贴近 Boss 职位列表 |
| JD 评分 | 普通卡片 | 「职位匹配度」风格：大号分数 + 环形/进度条 + 能力标签 pill |
| 预览页 | 单卡折叠 | 类「在线简历预览」：清晰分区、轻微分隔线、重点高亮 |
| 间距/密度 | 偏松 | 招聘站偏紧凑但清晰，统一 8px 栅格 |

## 二、改动范围（仅前端，不碰业务）

1. `frontend/src/App.vue`
   - `<style>` 全局样式重构（配色变量、顶栏、卡片、评分卡、预览、角标）
   - `<template>` 微调结构：顶栏改轻量导航；岗位选择改为标签/卡片化；JD 评分卡视觉升级
   - 不改动 `<script setup>` 中的任何状态/函数/数据流（保证逻辑字节级兼容）
2. `frontend/src/components/ResumeUploader.vue`
   - 上传拖拽区配色跟随新主题（绿系），去掉蓝链接色
3. `frontend/src/main.ts`
   - 注入全局 CSS 变量（可选，通过 `:root` 在 App.vue 统一即可，本计划优先改 App.vue 的 `<style>`）

## 三、不改动项（保持安全）

- `frontend/src/api/index.ts`：接口契约不变
- `frontend/src/components/InterviewAgentPanel.vue` 等面试组件：仅跟随全局样式变量，不改结构
- 后端 `backend/`：完全不涉及
- 所有 `<script setup>` 业务逻辑、字段名、emit 事件：保持不变（仅视觉层 CSS + 少量 template class/标签化）

## 四、关键视觉规范

- 主色：`--brand: #00bebd`（青绿），hover `#00a8a7`；辅色深灰 `#222` 文字、`#666` 次要、`#999` 占位
- 背景：`#f4f5f7`（招聘站常见浅灰底）
- 卡片：`background #fff; border:1px solid #ebedf0; border-radius:10px; box-shadow:0 1px 4px rgba(0,0,0,.04)`
- 圆角统一：`--radius:10px`
- 岗位标签（选中态）：绿底白字；未选：白底灰边
- JD 评分数字：大号 48px 加粗，颜色沿用现阈值色（绿/橙/红）逻辑，仅放大字号
- 步骤指示：简化为「选岗位 → 预览 → 下载」三段文字 + 当前态高亮，去除重型 el-steps 视觉负担（保留 el-steps 但换轻量样式，或直接用自绘步骤条）

## 五、执行顺序

1. 写计划文档（本文件）
2. 改 `main.ts`：在 `index.css` 引入前确保无冲突（可选）
3. 改 `App.vue` 全局样式 + 顶栏 + 岗位选择 + JD 评分卡模板微调
4. 改 `ResumeUploader.vue` 配色跟随
5. `cd frontend && npm run build` 验证类型/构建通过
6. 启动 dev 预览（或构建 preview）供用户查看

## 六、验收

- `npm run build` 成功，`vue-tsc` 无类型错误
- 顶栏、岗位选择、JD 评分卡呈现招聘站风格，逻辑功能不变
- 移动端响应式（sidecar → FAB）不受影响
